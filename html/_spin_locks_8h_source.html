118  /* \brief SpinRWMutex is a leightweight mutex that supports multiple readers (const-only access)
119  and a single writer. Supports readers up to the maximum value of size_t for your system.
120 
121  Calls to lock() from readers will block if a writer holds the lock. Calls to lock() from
122  writers will block if there are any readers or writers holding onto the lock.
123 
124  \note SpinRWMutex is easiest to use with SpinRWLock. SpinRWLock lets you "set-it-and-forget-it"
125  in regards to remembering if you're a reader/writer.
126 
127  \note SpinRWMutex does *NOT* check to ensure that calls to lock/unlock are called with the same
128  boolean. It does handle these cases (crash-wise), but calling lock(true) and unlock(false)
129  does not unlock the writer lock.
130 
131  Sample of a SpinRWMutex protecting a custom class MyClass:
132  \code
133 
134  mutable SpinRWMutex myRWMutex;
135  MyClass myClass;
136 
137  void setMyClass(const MyClass& other)
138  {
139  myRWMutex.lock(true); // true indicates a writer
140  myClass = other;
141  myRWMutex.unlock(true); // unlock my writer reference
142  }
143 
144  MyClass getMyClass() const
145  {
146  // static , thread-local object so it isn't constructed every function call, only copied
147  static thread_local MyClass ret;
148  myRWMutex.lock(false); // false indicates reader
149  ret = myClass;
150  myRwMutex.unlock(false); // unlock my reader reference
151  return ret;
152  }
153  \endcode
154  */
